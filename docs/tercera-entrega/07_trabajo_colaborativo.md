# 07 — Trabajo colaborativo

El desarrollo inicial se realiza de forma individual para establecer la base técnica.

## Incorporación posterior

Se incorporarán colaboradores para:

- Frontend y adaptación a mockups oficiales.
- QA, pruebas, evidencias, seguridad y documentación.

## Flujo de ramas

- `main` será la rama estable.
- `feature/*` se usará cuando se incorporen colaboradores.
- Todo cambio deberá pasar por Pull Request antes de merge.
- Está prohibido subir secretos, credenciales o configuraciones sensibles.
